# Phase 3.5 ‚Äî Storefront Design System & Frontend Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Apply the user's Calendly-style brand system across the storefront ‚Äî design tokens, reusable UI primitives, global header/footer chrome, a real landing page, and restyled catalog + (lightly) admin pages ‚Äî responsive and accessible, with the existing test suite kept green.

**Architecture:** Token-first. Brand values live once in `tailwind.config.ts` + `globals.css`; small presentational primitives in `components/ui/` consume them; pages compose primitives. Visual layer only ‚Äî no data-model, ingest, admin-logic, or pipeline changes.

**Tech Stack:** Next.js 15 (App Router, RSC), Tailwind CSS 3, `next/font` (Inter), React 19, Vitest + Testing Library.

## Global Constraints

- **Brand tokens (verbatim):** navy `#0B3558`, action-blue `#006BFF`, paper `#FFFFFF`, mist `#F8F9FB`, fog `#E7EDF6`, muted/slate `#476788`, ink/body `#0A0A0A`, border `#D4E0ED`; decorative gradients pink `#E55CFF`, purple `#8247F5`, amber `#FFA600`, cyan `#0099FF` (decoration ONLY ‚Äî never buttons/text/UI). Radius: button `4px`, card `16px`. Container `1200px`. Soft shadow `0 30px 50px rgba(71,103,136,.08)`. Spacing base 8px.
- **Tailwind color naming:** do NOT override Tailwind's built-in palette names (`blue`, `slate`, `green`, `gray`‚Ä¶) ‚Äî a single-hex override wipes the whole scale. Use distinct keys: `navy`, `action`, `paper`, `mist`, `fog`, `muted`, `ink`, `border`, `g-pink/g-purple/g-amber/g-cyan`.
- **Headings = navy, never pure black.** Body text = ink/near-black; secondary = muted. Primary buttons = action-blue, white text, 4px radius (NOT pill). Cards = paper, `border-border`, 16px radius, soft shadow.
- **Imagery = product screenshots / UI cards** (our layout previews) ‚Äî no stock photos.
- **Font: Inter** via `next/font/google` (Gilroy is proprietary; swappable later). If the build cannot fetch the font (no network), fall back to a system stack ‚Äî see Task 1 contingency.
- **Preserve tested behavior:** `LayoutCard` still links to `/layouts/<slug>` and shows the title; `Breadcrumbs` still renders crumb labels; `QueueTable` still renders rows + Approve/Reject + empty state. The full existing suite (95 tests) must stay green.
- **Visual layer only:** no changes to queries, ingest, admin actions/gating, or the pipeline. Buy buttons stay stubs (styled as primary CTAs ‚Üí `/pricing`).

// components/ui/Button.tsx
import Link from 'next/link';
import type { ReactNode } from 'react';

const base = 'inline-flex h-12 items-center justify-center rounded-button px-6 text-base font-semibold transition focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-action focus-visible:ring-offset-2 disabled:opacity-40';
const variants = {
  primary: 'bg-action text-paper hover:brightness-110',
  secondary: 'bg-paper text-navy border border-border hover:bg-fog',
};

export function Button({
  variant = 'primary',
  href,
  className = '',
  children,
  ...rest
}: {
  variant?: 'primary' | 'secondary';
  href?: string;
  className?: string;
  children: ReactNode;
} & React.ButtonHTMLAttributes<HTMLButtonElement>) {
  const cls = `${base} ${variants[variant]} ${className}`;
  if (href) {
    return <Link href={href} className={cls}>{children}</Link>;
  }
  return <button className={cls} {...rest}>{children}</button>;
}
